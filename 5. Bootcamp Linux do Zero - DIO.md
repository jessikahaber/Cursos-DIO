🔵 **Utilizando o WSL pelo Windows 10 ou 11:**

🔺 **Site Oficial da Microsoft**: [https://learn.microsoft.com/pt-br/windows/wsl/install](https://learn.microsoft.com/pt-br/windows/wsl/install)

🔺 O **WSL é um Subsistema do Windows para Linux que permite que os desenvolvedores instalem uma distribuição do Linux** (como Ubuntu, OpenSUSE, Kali, Debian, Arch Linux etc) **e usem aplicativos, utilitários e ferramentas de linha de comando bash do Linux diretamente no Windows**, sem modificação, sem a sobrecarga de uma máquina virtual tradicional ou configuração dualboot. Os desenvolvedores podem aproveitar o Windows e o Linux ao mesmo tempo em um computador Windows.

🔺 Antes de instalar **verificar se você preenche os pré-requisitos de instalação**, como: verifique se o seu sistema operacional é **Windows 10 pro ou 11 pro**. **Para checar qual o seu Windows**, clique no botão iniciar, depois digite e clique em “**sistema**”. O recurso “**Plataforma do Hipervisor do Windows**”, deve estar ativado também. E por ultimo, você também deve ativar o recurso “**Subsistema do Windows para Linux**”. Feito isso, aperte em “OK”, aguarde, e depois aperte em “Reiniciar agora”, ou “Não reiniciar” caso não possa desligar a máquina nesse momento.

🔺 **Após reiniciar o seu computador:** Você precisa fazer uma atualização do Windows, para isso, acesse o site: [https://learn.microsoft.com/pt-br/windows/wsl/install-manual](https://learn.microsoft.com/pt-br/windows/wsl/install-manual) . # **2.** ****Acesso Remoto a Máquinas Linux 🐧****

🔵  **Acesso Remoto via Windows**: 

🔺  **Na Virtual Box - Oracle:** Primeiro inicie a máquina virtual pelo Vitual Box da Oracle. Certifique-se de que o **hypervisorlaunchtype esteja off**. Faça o Login com usuário e senha.

1️⃣ Caso você precise **copiar um código que esta na sua máquina física para a máquina virtual**, se você tentar dar um ctrl + c e depois um ctrl + v, você pode ter um problema e não conseguir colar o código. Para isso, é necessário fazer um acesso remoto simulando que o **servidor Linux esteja em uma outra sala, ou em uma outra cidade**, para que você acesse remotamente essa outra máquina, então para isso **você precisará saber o endereço IP daquela máquina**. 

2️⃣ Para **saber todos os endereço de IP disponíveis no seu computador**, você precisa digitar o comando: **ip a** + enter. Você deverá buscar pelo “**enp0s3**”, a informação que consta ao lado de “**inet**”, os números xx.x.x.xx/xx, esses números representam o IP desta máquina. A partir de agora que **já sabemos o endereço ip desta máquina iremos realizar o acesso remoto dela pelo Windows**.

3️⃣ **Para realizar o acesso remoto da maquina virtual box através do Windows**, vamos utilizar um software chamado “**Putty**”, **para baixar esse software**, acesse o site: [https://www.putty.org/](https://www.putty.org/) , depois clique em “**Download PuTTY**”, desça a barra de rolagem até “****Alternative binary files****”, clique na opção “**putty.exe**” para realizar o download, eu fiz o do “**64-bit x86**”, esta opção esta disponivel em “**putty.exe (the SSH and Telnet client itself)**”, após fazer isso, desça novamente a barra de rolagem até “**puttygen.exe (a RSA and DSA key generation utility)**”, e clique na opção “**puttygen.exe**” para realizar o download, eu fiz o do “**64-bit x86**”, porque é a que esta de acordo com o meu sistema, verifique qual é o seu (menu iniciar + escrever sistema). 

4️⃣ **Atenção**: Você só conseguirá fazer o acesso remoto da sua MV **se a sua máquina física** (no meu caso Windows) **estiver na mesma faixa de IP da MV**, para saber disso, você precisa abrir a sua Prompt de Comando, digite o comando “**ipconfig** + enter”, desça a barra de rolagem até encontrar “**Adaptador de Rede sem Fio Wi-Fi:**”, e confirme se o IP que consta em “**Endereço IPv4**”, se consta na mesma faixa de IP, da MV, para que você consiga realizar acesso remoto a ela. 

**Exemplo**: IPv4 **10.0.0.**191 (máquina física) e inet **10.0.0.**103/24 (máquina virtual).

**Atenção**: Caso não esteja na mesma faixa de IP, **A rede da máquina virtual deve estar/ser configurada como** “**bridge**”. Para isso, certifique-se de que sua máquina virtual esteja desligada, clique em “**configurações**”, depois em “**rede**”, em seguida selecione a opção “**Habilitar placa de rede**”, Conectando a: “**Placa em modo bridge**”, no nome em baixo, escolha a opção que o seu computador físico esteja conectado a internet, pois é ela que a MV irá utilizar como ponte até a máquina física, então indique a placa de rede física do seu computador onde você tem acesso a internet, no meu caso é “**Adaptador de Rede 802.11n Broadcom**”, para verificar a sua, abra “**configurações de rede e internet**” na sua máquina física, depois clique em “**propriedades**”, agora baixe a barra de rolagem até identificar qual o seu caso, o meu esta constando na “**Descrição**”. Após isso selecionar a sua opção, aperte ok.

5️⃣ Agora execute o “PuTTY”, na parte escrita “**Host Name (or IP address)**”, digite o endereço IP da sua máquina virtual do Virtual Box, para que você consiga realizar o acesso remoto desta máquina. Exemplo: 192.168.1.102 . a “**Port**”, é a porta ”**22**”, esta é a padrão  de acesso remoto, do protocolo de “**SSH**”, agora aperte “**open**”. 

**Atenção**: **Caso dê algum erro**, seja pelo fato de ser necessário você instalar o serviço open SSH, para fazer isso, basta digitar na sua máquina virtual o comando: **apt-get install openssh-server** + enter. Caso você não esteja logado como usuário root, e apareça a mensagem que você não tem permissão para executar esse comando, digite: **sudo apt-get install openssh-server** + enter, digite a sua senha, depois digite que deseja continuar a instalação.

Agora deve aparecer a mensagem (**PuTTY Security Alert**):

**The host key is not cached for this server:**

**192.168.1.102 (port 22)**

**You have no guarantee that the server is the computer you think it is.**

**The server's ssh-ed25519 key fingerprint is:**

**ssh-ed25519 255 SHA256:FhLOFBdrZjVOV9eJ5PcQWj2OLZkcS7Md0ezGdT0EdH8**

**If you trust this host, press "Accept" to add the key to PuTTY's cache and carry on connecting.**

**If you want to carry on connecting just once, without adding the key to the cache, press "Connect Once".**

**If you do not trust this host, press "Cancel" to abandon the connection**

Clique em “**Accept**”, com isso você estará concordando que está fazendo acesso remoto. **Agora digite o seu login e senha da máquina virtual** que você escolheu quando realizou a instalação do Ubuntu. Pronto, agora você está acessando uma máquina que pode estar em outra sala, outra Cidade, Estado ou Pais.

**Observação**: para se conectar a uma máquina na mesma faixa de IP, você precisa garantir que as máquinas estejam conectadas à **mesma rede e que o endereço IP da máquina remota esteja correto**. Além disso, é importante que **a máquina remota tenha um servidor SSH instalado, configurado e em execução para permitir a conexão**.

# 3. ****Manipulando Arquivos no Linux 🐧****

🔵 **Comandos básicos no Linux - Navegando pelo sistema de arquivos 🐧:**

🔺 **reboot** + enter = irá **reiniciar a máquina/computador**. Atenção: Caso apareça que você não tem permissão para isso, escreva o comando: **sudo reboot** + enter . Tudo em minúsculo.

🔺 **ip a** + enter = Para **saber todos os endereço de IP disponíveis no seu computador**.

🔺 **clear** + enter = Para limpar a tela de comandos, ou pode pressionar a tecla: **ctrl + l** , que irá limpar a tela de comando da mesma forma.

🔺 **pwd** + enter = Para saber onde você esta dentro da máquina.

🔺 **cd /** + enter = Para se mover entre os diretórios. Esse comando leva direto para a **/** que é a raiz de todos os diretórios do Linux. **Atenção**: **/** é o diretório raiz do Linux e **\** é o diretório raiz do Windows.

🔺 **ls** + enter = Lista todos os arquivos e diretórios que estão dentro daquele diretório que eu estou. Todos os arquivos em azul escuro são diretórios.

🔺 **cd /var** + enter = Serve para você entrar dentro do diretório var, você estando ou não dentro do diretório raiz.

🔺 **cd var** + enter **=** Você entra dentro do diretório var, para entrar em outro diretório basta substituir a palavra var, pelo nome do diretório que você deseja entrar. ****

🔺 **cd ..** + enter **=** Volta um diretório atrás, ou seja, sai do diretório que você está e volta pro diretório anterior.

🔺 **cd ../nomedodiretório** + enter = volta um diretório e já abre um novo diretório de lá. **Exemplo**: **cd ../log** + enter.

🔺 **cd /var/log** + enter = Você vai da raiz direto para dentro do diretório log, esse comando serve para acessar outros diretórios também caso você já saiba o caminho, basta usar o / que fará a ponte entre um diretório e outro.

🔺 **cd /var/** + TAB TAB= vai te mostrar/listar os diretórios que estão dentro do diretório var, caso você tenha esquecido os diretórios que estão ali dentro, e o comando continuará lá para você complementar com o diretório que você deseja acessar. **Dica**: Após escrever o segundo diretório ex: cd /var/log/ + tab tab (**aperte tab duas vezes**), vai listar novamente os diretórios que estão dentro do diretório log, e assim consecutivamente, até que chegue onde deseja acessar. Esse comando tem a mesma lógica para todos os diretórios, não somente no var. **Atenção**: Nesse comando quando o diretório que você deseja entrar só tiver **um diretório ali dentro**, ele já vai entrar automaticamente, para ter certeza, basta voltar um diretório e digitar o comando ls para listar.

🔺 **cd una** + tab = preenche o nome de um arquivo que você começou a escrever, apertando a tecla tab ele termina de escrever sozinho o nome do diretório que tem ali dentro com as letras iniciais que você já escreveu. **Atenção**: Se houver outros arquivos com essa inicial, você deve escrever mais letras daquele arquivo ou pressionar tab duas vezes e irá listrar os arquivos que possuem aquelas letras iniciais.

🔵 **Comandos básicos no Linux - Filtrando a exibição de arquivos:**

🔺 **ls | more** + enter = Quando dentro de um diretório lista vários arquivos e diretórios e não aparecem todos na tela, e não há barra de rolagem, ao você digitar esse comando, ele faz uma lista de todos os arquivos com a opção “more” no final, com isso, conforme você vai apertando enter, ele vai rolando de vagar para baixo mostrando o restante dos arquivos e diretórios que se encontram ali dentro daquele diretório que você listou. **Dica**: **Para sair dele cancelando a exibição, basta pressionar ctrl + c**

🔺 **ls nomedoarquivo** + enter = você faz uma busca para saber se tem algum arquivo com aquele nome na lista de arquivos. **Exemplo**: ls passwd + enter. Se tiver, ele será exibido a baixo. Se não existir, apresentará uma mensagem de que não existe esse arquivo ou diretório.

🔺 **ls iniciaisdoarquivo** + tab = preenche o nome do arquivo caso só exista ele, se houver outros com essa inicial, aperte tab duas vezes, e irá aparecer abaixo uma lista com arquivos e diretórios com as iniciais que você digitou.

🔺 **ls p*** + tab = vai listar todos os arquivos e diretórios que iniciam com a letra p, esse comando serve para as outras letras do alfabeto também, basta colocar a inicial do arquivo que você busca. A vantagem de usar o * do lado da inicial, é que além de listar todos os diretórios e arquivos com a inicial que você escreveu, ele também mostra todos os arquivos que estão dentro de cada diretório. **Atenção**: o * significa que o restante do nome do arquivo ou diretório pode ser complementado por qualquer coisa, ou seja, possuir qualquer caractere.

🔺 **ls m?g*** + enter = vai buscar um arquivo que a primeira letra seja m, a segunda letra, pelo uso do ? pode ser qualquer letra ou número, a terceira será obrigatoriamente a letra g, e a quarta em diante pelo uso do * pode ser qualquer coisa, ou seja, possuir qualquer caractere. É a mesma lógica para a busca de outros arquivos, bastando substituir as letras. Se tiver o arquivo, ele irá listar abaixo. Outro **exemplo**: **ls ?sh*** + enter. Significa que a primeira letra pode ser qualquer uma. Se existir algum arquivo assim ele irá listar.

🔺 **touch arquivo1.txt** + enter = irá **criar um arquivo vazio** no formato .txt. com o nome arquivo1. **Atenção**: Observe qual diretório você esta, porque ao digitar esse comando, ele irá criar esse arquivo dentro do diretório que você estiver dentro. **Dica**: Se aparecer uma mensagem dizendo que você não tem permissão basta acessar como usuário root.

🔺 **cd ~** + enter = se vc estiver na raiz /, você irá direto para o diretório do usuário.

🔺 ⬆️ ⬇️ = o uso das setas no teclado para cima e pra baixo, você consegue usar para ir voltando aos comandos que você usou anteriormente, assim você não precisa digitar tudo novamente quando precisar repetir um comando.

🔺 **ls arquivo[1-3].txt** ou **ls arquivo[1-3]*** + enter = irá listar os arquivos que possuem esse nome do 1 ao 3, o primeiro comando listara só em formato .txt e o segundo comando em qualquer formato. Esse comando é bom para fazer cópia dos arquivos que possuem determinada frequência númerica. **Atenção**: onde esta escrito arquivo, pode ser substituído pelo nome do arquivo. **Dica**: se colocar **[2-5]** irá listar do 2 até o 5. se você colocar **[2,5]**, irá listar somente o 2 e o 5. se colocar **[^2,5]**, irá listar todos os arquivos, exceto o 2 e o 5. se colocar **[^3-5]**, irá listar todos os arquivos, exceto do 3 até o 5.

🔵 **Comandos básicos no Linux - Localizando arquivos:** 

🔺 **Buscar arquivo sem ter que entrar dentro do diretório de busca**: Você não precisa necessariamente estar dentro do diretório para realizar uma busca, basta indicar qual diretório você quer que seja realizada a busca. **Ex: ls /nomedodiretório + enter**. **Ex2: ls /sys + enter**. Irá exibir o que esta dentro do diretório sys, sem necessariamente você ter que entrar dentro dele para listar, ou seja, da raiz você consegue fazer isso. **Ex3: ls /sys/kernel + enter**. É a mesma lógica, irá mostrar o que tem dentro de kernel sem você precisar estar lá dentro, estando na raiz você consegue realizar essa busca.

🔺 **Buscar arquivo sem ter que entrar dentro do diretório de busca filtrando por uma letra específica**: **Ex: ls /sys/kernel/p* + enter** . Irá mostrar todos os diretórios e arquivos dentro de kernel que iniciam com a letra p, como utilizo * após a letra, significa que o restante pode ser qualquer caractere.

🔺 **Buscar arquivo sem que você saiba onde ele está no seu computador:** Você sabe que o arquivo esta no seu computador mas não sabe o diretório principal, então você pode realizar uma busca a partir da raiz de todos os diretórios e arquivos. Irá criar um diretório a partir da raiz / com o nome de Documentos. **Atenção**: Caso apareça a mensagem que você não tem permissão para isso, basta você logar como super usuário (root). **Exemplo2**: Entre dentro do seu diretório em home, e digite o comando: **mkdir nomedodiretório** + enter.  **Exemplo: mkdir Documentos** + enter. Assim **irá criar dois diretórios de uma vez em apenas um comando**, é a mesma lógica para a quantidade de diretórios que você pretende criar, bastando dar um espaço e digitar o nome do diretório.

🔵 **Comandos básicos no Linux - Excluindo arquivos e diretórios:** 

🔺 **Para excluir um diretório**: Utilize o comando: **rmdir nomedodiretório** + enter. **Exemplo: rmdir ‘Ola Jessika’** + enter. E com isso será excluido o diretório ‘Ola Jessika’. **Atenção 1**: **Caso você tenha conteúdo dentro do diretório que pretende excluir**, e você digitar esse comando, irá aparecer uma mensagem dizendo que o diretório não está vazio, então você terá que limpar/excluir o que tem lá dentro, para depois poder conseguir excluir o diretório. **Atenção 2**: Você não consegue excluir um diretório estando dentro dele.

🔺 **Para excluir vários diretório**: Caso tenha uma serie de diretórios que você pretende excluir de uma só vez, como por exemplo, Escolares e Financeiras, dois diretórios diferentes, você pode digitar o comando: **rmdir nomedodiretório nomedodiretório** + enter. **Exemplo: rmdir Escolares Financeiras** + enter. Assim irá excluir os dois diretórios com apenas um comando, se ambos estiverem vazios. **Atenção**: Para excluir vários diretórios de uma só vez, mesmo que eles tenham conteúdos e sub-conteúdos dentro de si, utilize o comando: **rm -rf nomedodiretórioquevocêquerapagar** + enter. **Exemplo: rm -rf Planilhas** + enter. A letra **f**, depois do r, significa que você **irá apagar os diretórios e os arquivos** que estão dentro do diretório principal que você deseja apagar. Dessa forma você irá apagar o diretório Planilhas e todo o conteúdo que estiver ali dentro, sejam diretórios ou outros arquivos.

🔺 **Para excluir um arquivos .txt:** Utilize o comando: **rm nomedoarquivo.txt** + enter. **Exemplo: rm texto1.txt** + enter. Com isso irá excluir o arquivo texto1.txt.

🔺 **Para excluir vários arquivos .txt:** Caso tenha uma serie de arquivos que pretende excluir de uma só vez, como o caso de, arquivo1.txt, arquivo2.txt, arquivo3.txt, arquivo4.txt, arquivo5.txt, você pode digitar o comando: **rm arquivo*** + enter. Assim irá excluir todos os arquivos que comecem com arquivo, e o restante pode ser qualquer caractere por conta do uso do *. **Exemplo 2: rm *.txt** + enter. Irá excluir todos os arquivos que em seu nome contenha qualquer caractere, e que a extensão é em .txt.

🔵 **Comandos básicos no Linux - Executando tarefas administrativas como root:**

🔺 **Caso você precise realizar uma atividade de administrador**: Como por exemplo: Instalar um programa, criar um diretório em um lugar específico, e etc. **Atenção**: Se você criar um novo usuário a partir de um que já existia, para que ele possa realizar essas tarefas de administrador, ele precisa antes, ser adicionado no grupo de administradores.

🔺 **Como checar os grupos de administradores**: Os grupos de administradores estão armazenados dentro de um arquivo de texto no Linux, então você consegue visualizar facilmente. 

🔺 **Para visualizar o conteúdo de um arquivo de texto**: Você precisa digitar o comando: **cat /nomedodiretórioqueoarquivoemtextoesta/nomedoarquivoquevocêquervisualizar** + enter. **Exemplo: cat /etc/group** + enter. Com isso será **listado todos os grupos que você tem no sistema**. Dentro de **adm** e **sudo**, estará listado alguns **usuários que podem realizar atividades de administrador**.

🔺 **Executando tarefas administrativas como root:** Neste exemplo vamos realizar a **tarefa de criar um diretório na raiz, estando dentro da raiz**. Para isso, digite o comando: **sudo mkdir Aulas** + enter. Ao usar o comando sudo, você está realizando o comando como super usuário (root), logo esse diretório que você criou pertence ao root, sendo assim, para criar arquivos lá dentro, ou diretórios, ou fazer alguma alteração, você só poderá fazer como super usuário (root), utilizando o comando sudo antes de qualquer outro comando.

🔺 **Arquivo de configuração para verificar se determinado usuário pode ter atividade de administrador**: Para verificar esse arquivo, digite o comando: **sudo cat /etc/sudoers** + enter. É neste arquivo que esta as configurações do comando sudo, quais membros possuem o acesso como root. No grupo admin, são todos os privilégios. No grupo sudo os usuários que estão ali dentro também podem executar comandos como Administrador.

🔵 **Comandos básicos no Linux - Logando como usuário root:**

🔺 **Para quê serve logar como usuário root**: Serve para não ter que ficar escrevendo o comando sudo e/ou ficar colocando senha sempre que executar um comando. Você pode logar como super usuário, realizar as tarefas administrativas e depois voltar para o usuário normal. **Dica**: Não é aconselhável ficar sempre logado como root. **Atenção**: No Ubuntu, o usuário root já esta criando, mas se você quiser logar como usuário root, você precisa criar uma senha para ele.

🔺 **Criar uma senha para usuário root**: Para criar essa senha, você precisa digitar o comando: **sudo passwd nomedousuárioquevocêvaidarasenha** + enter. **Exemplo: sudo passwd root** + enter. Agora digite a senha do usuário antes de atribuir uma senha para o usuário root e aperte enter, agora sim, digite a senha para o root e aperte enter, repita a senha e pronto, você criou a senha para o usuário root. **Atenção**: O comando passwd atribui uma senha há um usuário já existente. **Dica**: Escolha uma senha complexa para o usuário root, porque quem possui esse acesso poderá fazer de tudo, até excluir todo o seu sistema.

🔺 **Logando como usuário root**: Para realizar o login do usuário root, digite o comando: **su** + enter. Depois digite a senha do Administrador root. Agora você pode observar que ao em vez do símbolo **$**, estará um **#**, isso indica que você esta logado como super usuário. Então a partir de agora você não precisa usar o comando sudo para nada.

🔺 **Saindo do usuário root e voltando para o usuário convencional**: Para isso digite o comando: **su nomedousuário** + enter. **Exemplo**: **su jessika** + enter.
